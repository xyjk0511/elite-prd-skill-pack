# PRD Audit Rubric

Score each dimension from 0 to 5.

| 维度 | 满分 | 判断标准 |
|---|---:|---|
| 问题定义 | 5 | 是否说清楚为什么做 |
| 目标用户 | 5 | 是否具体到角色与场景 |
| 产品目标 | 5 | 是否明确结果和价值 |
| 成功指标 | 5 | 是否可量化或可观察 |
| 范围边界 | 5 | 是否明确 in scope / out of scope |
| 用户流程 | 5 | 主流程、分支流程、异常流程是否完整 |
| 功能描述 | 5 | P0/P1 功能是否有足够规则 |
| 业务规则 | 5 | 规则是否编号、明确、可测试 |
| 状态流转 | 5 | 关键对象是否有状态定义和流转规则 |
| 权限设计 | 5 | 多角色权限是否清楚，后端校验是否明确 |
| 字段规则 | 5 | 必填、类型、限制、错误提示是否齐全 |
| 异常处理 | 5 | 网络、权限、非法输入、重复提交、冲突是否覆盖 |
| 验收标准 | 5 | 是否可测试，是否覆盖 P0 功能 |
| 数据埋点 | 5 | 是否定义事件、属性、漏斗和失败原因 |
| 非功能需求 | 5 | 性能、安全、隐私、兼容性、可靠性是否覆盖 |
| 上线方案 | 5 | 是否有灰度、回滚、监控或复盘 |
| 风险依赖 | 5 | 是否能用于排期和评审 |

Scoring:

- 5：完整、明确、可执行、可测试。
- 4：基本完整，只有少量细节需要补充。
- 3：方向清楚，但研发/测试仍需多次追问。
- 2：有明显缺口，不能直接开工。
- 1：只有想法，没有形成需求。
- 0：缺失或不可判断。

Readiness:

- 75-85：可以开工。
- 55-74：补充后开工。
- 0-54：暂不建议开工。

Any missing P0 acceptance criteria, unclear state machine for stateful objects, or missing sensitive permission rule should be treated as a blocker even if the numeric score is high.
