# PRD Template

Use this template for full PRDs unless the repository has a stronger existing template.

```md

## 1. 文档信息

| 字段 | 内容 |
|---|---|
| 文档名称 |  |

| 版本 | V1.0 |
| 作者 |  |
| 创建日期 |  |
| 最后更新 |  |
| 状态 | 草稿 / 评审中 / 已确认 / 开发中 / 已上线 |
| 相关人员 | 产品 / 设计 / 前端 / 后端 / 测试 / 数据 / 运营 |

## 2. 一句话概述

为了谁，解决什么问题，提供什么能力，达成什么结果。

## 3. 背景与问题

### 3.1 业务背景
### 3.2 用户问题

### 5.1 产品目标
### 5.2 用户目标

### 5.4 成功指标
### 5.5 反指标 / 护栏指标

### 6.1 本期范围
### 6.2 非本期范围

### 6.4 优先级说明

## 7. 用户流程

### 7.1 主流程
### 7.2 分支流程
### 7.3 异常流程
### 7.4 退出与返回流程

## 8. 页面与模块结构

### 8.1 页面列表
### 8.2 页面入口
### 8.3 页面出口
### 8.4 模块说明
### 8.5 交互状态

| 状态 | 触发条件 | 页面/组件表现 | 用户可操作项 | 系统处理 |

- 业务规则：

- 验收标准：
- 埋点需求：

## 10. 业务规则

| 规则编号 | 规则内容 | 影响范围 | 备注 |
|---|---|---|---|

## 11. 状态流转

### 11.1 状态定义

| 状态 | 含义 | 用户可见 | 可操作项 | 进入条件 | 退出条件 |
|---|---|---|---|---|---|

```

|---|---|---|---|---|

## 12. 权限设计

| 操作 | 未登录用户 | 普通用户 | 管理员 | 运营 | 备注 |
|---|---|---|---|---|---|

敏感操作必须进行后端权限校验。

## 13. 字段与数据要求

| 字段 | 类型 | 是否必填 | 默认值 | 限制 | 示例 | 校验时机 | 错误提示 |
|---|---|---|---|---|---|---|---|

## 14. 接口与系统交互要求

说明前后端、第三方服务、内部系统之间的交互需求；不要过度绑定内部实现。

## 15. 数据埋点与分析

### 15.1 成功指标

| 指标 | 定义 | 口径 | 目标值 | 观察周期 |
|---|---|---|---|---|

### 15.2 事件埋点

|---|---|---|---|

## 16. 非功能需求

性能、安全、隐私、兼容性、可用性、稳定性、可维护性。

## 17. 异常与边界情况

覆盖网络失败、权限不足、输入非法、重复提交、数据为空、状态冲突等。

## 18. 验收标准

按功能、流程、边界、权限、数据、性能列出验收条件。

## 19. 测试用例建议

给 QA 可以直接展开的测试场景。

|---|---|---|---|---|---|

| 依赖方 | 依赖内容 | 截止时间 | 风险 | 状态 |
|---|---|---|---|---|

## 22. 待确认问题

## 24. 附录

原型、竞品、术语、参考资料、历史版本。
```

For compact PRDs, keep at least: 需求概述、背景与问题、目标用户、产品目标、成功指标、本期范围、非本期范围、用户流程、核心功能、业务规则、异常情况、验收标准、埋点需求、风险与待确认问题。
