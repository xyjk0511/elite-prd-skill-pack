# PRD Quality Checklist

Use before final output.

| 检查项 | 通过标准 |
|---|---|
| 问题定义 | 明确写清楚为什么做 |
| 目标用户 | 不是“所有用户”，而是具体角色 |
| 产品目标 | 可以衡量或可以被验收 |
| 成功指标 | 有口径、目标值或观察方式 |
| 范围边界 | 有本期范围和非本期范围 |
| 用户流程 | 包含主流程、分支流程和异常流程 |
| 功能需求 | P0 功能有 FR 编号与规则 |
| 状态 / 权限 | 对象状态与权限矩阵清晰 |
| 字段规则 | 必填、长度、范围、错误提示齐全 |
| 验收标准 | 可测试，不含模糊词 |
| 数据埋点 | 有事件名、属性和漏斗 |
| 非功能需求 | 覆盖相关性能、安全、隐私、兼容性、可靠性 |
| 风险与依赖 | 能用于评审与排期 |
| 待确认问题 | 未决问题单列，不埋在正文里 |

Silent self-review:

1. 设计能否直接出原型。
2. 前端能否识别页面、交互状态、校验规则。
3. 后端能否识别对象、接口、权限、状态机。
4. QA 能否写测试用例。
5. 数据能否做埋点。
6. 是否清楚写明本期不做什么。

If any answer is no, fix the PRD before returning it.
